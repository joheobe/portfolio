Explanation of the Professional Resume HTML Code

1. Document Structure and Head Section

DOCTYPE and HTML Tag: The code starts with <!DOCTYPE html> to define the document as HTML5. The <html lang="en"> tag specifies that the language is English.

Meta Tags:

  • <meta charset="UTF-8"> sets the character encoding to UTF-8.
  • <meta name="viewport" content="width=device-width, initial-scale=1"> ensures the layout is responsive on various devices.

Title: The <title> tag sets the text displayed in the browser tab.

Font Awesome: A <link> tag includes the Font Awesome CDN, enabling the use of icon fonts in the resume.

Internal CSS: A <style> block contains all the CSS for layout, typography, and styling of components like cards, grids, and progress bars.

2. Global Styles

The CSS for the body sets the default font to Arial, sans-serif, removes default margins, applies a light grey background (#f1f1f1), and increases the base font size to 18px for better readability. Additionally, link styles are defined to have a teal color with underlining on hover.

3. Container and Grid Layout

The .container class centers the content with a maximum width of 1400px and adds padding. The .row class uses Flexbox to arrange child elements in a row with wrapping enabled.

Two column classes are used:

  • .column-left: occupies 33.33% of the width.
  • .column-right: occupies 66.66% of the width.
A media query ensures that on screens smaller than 800px both columns expand to 100% width.

4. Card Component

The .card class creates a white background card with rounded corners, a subtle shadow for depth, and a margin at the bottom. The .card-content class adds padding to the content within each card.

5. Image and Name Overlay

The .image-container is set to position: relative so that its child elements can be positioned absolutely. The profile image fills 100% of the container’s width.

The .name-overlay is positioned at the bottom left of the image with a semi-transparent white background. Its font size is increased to 32px and set to bold, making the name “Alex Johnson” stand out.

6. Headings and Text Styling

Different heading tags are used to structure the content:

  • h2: Used for main section titles (e.g., Professional Experience) with a font size of 28px and teal color.
  • h3: Used for sub-sections like Technical Skills and Languages, with a font size of 24px.
  • h5: Typically for job titles or institution names, set at 20px in grey.
  • h6: Used for dates or additional details, set at 16px in teal.
Icons included in headings and paragraphs have extra right margin for spacing.

7. Horizontal Rules

The <hr> elements are styled as thin, light grey lines (1px solid #eee) with margins to separate sections.

8. Progress Bars for Skills and Languages

Each progress bar is built using two nested divs:

  • .progress-container: Acts as the track with a light grey background and rounded corners.
  • .progress-bar: The inner bar with a teal background and white text that displays the percentage. Its width is set inline (e.g., width:95%) to represent proficiency.

9. Footer

The footer has a teal background with white text, centered with padding. It includes social media links (LinkedIn, GitHub, WhatsApp) styled with larger font sizes for the icons, and a copyright notice.

10. Body Content

The page is divided into two main columns: Left Column:

  • Contains a card with the profile image and name overlay.
  • Displays contact information with icons for job title, location, email, and phone.
  • Lists technical skills and languages with corresponding progress bars.
Right Column:
  • Includes a card for Professional Experience with multiple job entries (job title, date range, description).
  • Includes a card for Education with details about academic credentials.

Summary

This HTML and CSS code creates a modern, responsive professional resume layout using Flexbox for the grid, card components for separate sections, progress bars to represent skills, and Font Awesome for icons. Responsive design techniques ensure the resume looks good on both desktop and mobile devices.

To connect your GitHub account to VS Code and publish your HTML/CSS code live, follow these steps:

  1. First, link your GitHub account to VS Code. To do this, click on the account icon at the bottom-left corner of the VS Code window. Then, select "Backup and Sync Settings" from the dropdown menu.
  2. In the VS Code search bar, type "Sign In" and click on the result. This will prompt you to sign in to your GitHub account.
  3. Next, install the "Git Pull" extension from the VS Code Extensions marketplace.
  4. Initialize a new Git repository in your project folder. Make sure to set the repository visibility to "Public".
  5. Commit and sync your changes to GitHub.
  6. Go to your GitHub account and navigate to the repository you just created. Click on "Settings", then scroll down and select "Pages" to publish your HTML/CSS code live.
  7. In the GitHub Pages section, select "main" or "master" as the branch, and choose the root directory as the source.
  8. Click "Save".
  9. Wait a few minutes for GitHub to publish your site.
  10. You should now see your site live at the provided GitHub Pages URL.
